# 스내사 3기 대시보드 제품 개발 계획안 (이미지 기반 인증 대시보드)

## 제품 방향

스내사 3기 대시보드는 러닝 기록 자동 연동 기능을 제거하고, 운영자가 참가자들의 러닝 기록 이미지를 업로드해 인증 여부와 향상도를 관리하는 대시보드형 제품으로 전환한다.

## 핵심 사용자

- 러닝 챌린지 운영자
- 10~30명 규모의 참가자 인증을 매일 확인해야 하는 관리자
- 이미지 인증 자료를 기반으로 거리/시간 변화를 보고 싶은 코치 또는 리더

## 핵심 플로우

1. 운영자가 참가자를 직접 등록한다.
2. 운영자가 날짜를 선택하고 참가자들이 보낸 러닝 인증 이미지를 업로드한다.
3. AI가 이미지에서 배경을 무시하고 텍스트/숫자만 추출한다.
4. 이름, 날짜, 거리, 시간이 자동 매칭된다.
5. 날짜나 시간이 없으면 `확인 필요`로 남기고 운영자가 직접 입력한다.
6. 대시보드에서 인증 시계열, 거리/시간 그래프, 랭킹을 확인한다.

## MVP 구현 범위

- 참가자 등록
- 이미지 일괄 업로드
- Gemini Vision 기반 OCR/구조화 추출
- 날짜 누락 시 기본 날짜 임시 적용
- 시간 누락 시 수동 입력
- 참가자 매칭 실패 시 수동 선택
- 인증/확인 필요/반려 상태 관리
- 최근 14일 인증 시계열
- 인증 횟수/거리/시간 랭킹
- 개인별 거리/시간 그래프

## 주요 테이블

- `participants`
- `upload_batches`
- `daily_run_records`

## 개발 단계

### Phase 1: 데이터 구조

- Supabase 스키마 정리
- 참가자/기록/업로드 배치 테이블 추가
- RLS 정책 추가

### Phase 2: OCR 파이프라인

- 이미지 base64 업로드
- Supabase Storage 저장
- Gemini Vision으로 텍스트/숫자 추출
- JSON 구조화
- 참가자 자동 매칭
- 검수 필요 상태 판정

### Phase 3: 운영 대시보드

- 요약 카드
- 참가자 추가
- 이미지 업로드
- 수동 기록 입력
- 검수 테이블
- 인증 시계열
- 랭킹
- 개인별 그래프

### Phase 4: 운영 안정화

- 중복 이미지 감지
- OCR 신뢰도 기준 조정
- CSV 내보내기
- 월간 리포트
- 참가자별 상세 페이지
